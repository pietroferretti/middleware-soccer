Starts the parser, which receives events from all sensors and communicates with the output and possession processes.

Start, end and interruptions of the game are highlighted in the data received by the process.

Parameters
mpi_position_for_possession_typeMPI datatype used to send messages to the possession process
mpi_output_envelopeMPI datatype used to send messages to the output process
possession_processesNumber of possession processes that are running, used to know buffer sizes
TLength of time between outputs
fullgame_pathPath for the position events file
interr_path_onePath for the first game interruptions file
interr_path_twoPath for the second game interruptions file
